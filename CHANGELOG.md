## Unreleased

### Added
- **`Makefile` with one-liner install and packaging targets.** `make install` copies every skill folder under `_skill-sources/` into `~/.claude/skills/` (overridable with `SKILLS_DIR=/some/path`), replacing any previous copy. `make package` rebuilds every `<skill>.skill` archive at the repo root from the corresponding source folder. `make clean-archives` removes them. `make help` (the default target) lists everything. README install step now leads with `make install` and keeps the explicit `cp -r` block as a fallback for users without `make`; the Personalization section points at `make install` for re-pushing edits and `make package` for rebuilding archives. Closes the remaining piece of ROADMAP 1.5.
- **Skip-zone marker convention.** Three HTML-comment marker pairs let writers hide passages from editorial review without affecting how the manuscript renders in Obsidian:
  - `<!-- quill: skip-start -->` … `<!-- quill: skip-end -->` — Quill only.
  - `<!-- warden: skip-start -->` … `<!-- warden: skip-end -->` — Warden only.
  - `<!-- skip-start -->` … `<!-- skip-end -->` — Atlas, Quill, Warden (the typical choice for in-world documents like letters or articles).

  Inside a skipped region, the affected skills suspend all of their checks (Warden's timeline claims and Atlas's provisional-voice derivation included). Lens deliberately does *not* honor skip markers — readers see whatever is on the page. Parsing rules are identical across the three honoring skills: markers on their own line, nearest-match start-to-end, no same-type nesting, malformed markers flagged in the report without failing the run. Quill and Warden record `_Skip zones honored: N_` in the report header (omitted when zero); Atlas records the same in its `atlas_history.md` session entry. `atlas.skill`, `quill.skill`, `warden.skill`, `lens.skill` archives rebuilt. `USAGE.md` recipe and Conventions entries added. (ROADMAP 3.9)
- **`USAGE.md` — Writer's Handbook.** New top-level user-facing handbook aimed at amateur users. Plain-language walkthrough of a first session; per-skill writer's view of all six skills with what to expect in reports and what the skill won't do; conventions reference (provisional markers, severity tiers, review modes, empty-Atlas fallback lines, paragraph references); recipe section for common situations ("I just finished Chapter 3 — what now?", "Quill is too noisy", "I changed my mind about a character's eye color"); glossary; pointers to ROADMAP / CHANGELOG / individual `SKILL.md` files. Back-fills writer-facing detail for everything shipped so far (6.6 path-diagram fix, 1.6 `/new-project` polish, 2.6 Warden severity + timeline, 2.7 Quill review modes, 3.8 empty-Atlas fallbacks + provisional voice). README updated with a link to it and a new entry in the "What's inside" table. (ROADMAP 6.7)
- **Atlas provisional voice derivation.** When a character appears in two or more chapters with dialogue / interior monologue and their Voice / Speech-marker fields are empty, Atlas may derive a provisional profile from observable patterns (vocabulary register, sentence-length tendencies, contractions, recurring constructions — never personality inference). The profile carries the `[provisional, derived from Ch.X–Y]` marker. Atlas extends the profile across runs but never overwrites it once the writer edits the marker out (the upgrade-to-permanent path). Voice is the only field eligible for provisional derivation; every other Atlas field is recorded literally or stays empty. (ROADMAP 3.8)
- **Empty-Atlas fallback rules in Quill, Warden, and Lens.** Consumer skills now explicitly state which checks were skipped because Atlas had no data, with concrete report-line phrasings (e.g. `Character consistency: skipped — no Characters in Atlas.`). Lens falls back to direct reading when its navigation index is missing rather than becoming an Atlas reader. Quill and Warden distinguish provisional voice profiles from permanent ones and flag drift conservatively against provisional baselines. `atlas.skill`, `quill.skill`, `warden.skill`, `lens.skill` archives rebuilt. (ROADMAP 3.8)
- **Quill review-depth modes.** Three modes — `drafting` (default), `balanced`, `polish` — control how much Quill surfaces. `drafting` keeps Major Suggestions + Voice Notes + All Clear and suppresses Minor Notes / Phrasing / Vocabulary; `balanced` adds capped Minor / Phrasing / Vocabulary; `polish` opens the firehose. Mode resolves in priority order: per-invocation argument → Ledger pipeline mode (forward-referenced to ROADMAP 2.1) → `_meta/quill/config.yml` → default. Report header records the active mode and its source. `quill.skill` archive rebuilt. (ROADMAP 2.7)
- **Warden severity tier `Question`.** Third severity alongside Major and Minor, for cases where a discrepancy *could* be a slip or *could* be intentional (foreshadowing, unreliable narrator, character lying, alt-take). Warden cannot tell which from text alone — Question hands the call to the writer. Inline schema updated; severity guide rewritten with the new tier and a forward reference to ROADMAP 2.5 (acknowledgement convention). `warden.skill` archive rebuilt. (ROADMAP 2.6)
- **Warden `Timeline` section.** New optional section in Warden reports listing travel-time, date, season, and duration claims observed in the file, cross-referenced against Atlas / prior chapters. Side-by-side tables make contradictions visually obvious. Section is omitted when the chapter contains no such claims — pure presentation, no new detection logic. (ROADMAP 2.6)
- **`/new-project` skill.** Scaffolds a new project folder under `01_Projects/` from `templates/`. Asks for title and project type (novel, standalone short, or shared-universe collection), confirms the planned paths, then creates `_meta/atlas.md`, `_meta/status.md`, and an empty `Lore/` folder where appropriate. Does not generate prose, premises, or placeholder content. Hands off to `/atlas` for first-run map initialization. (ROADMAP 1.2)
- **`templates/` directory.** Optional-everything stub files for `_meta/atlas.md`, `_meta/status.md`, and `Lore/` notes (character, location, world-rule, timeline). Every field in every template is optional — they describe what Atlas *can* track, not what writers must fill in. (ROADMAP 1.3)
- **Editorial-only guardrails block at the top of every skill.** Each `SKILL.md` now opens with a "read first" rules block that overrides later instructions: no prose generation, structured output only, ownership boundaries, and a stop-and-ask escape hatch when a request would violate the rules. Hardens the editorial boundary mechanically rather than behaviorally. (ROADMAP 1.1)
- **`CHANGELOG.md`.** This file.
- **Core principles section in `CLAUDE.md`.** Codifies editorial-only, writer-led, and minimal-by-default as project-wide directives.
- **`_meta/` convention section in `CLAUDE.md`.** Clarifies that `_meta/` files are agent-owned and should not be hand-edited.
- **Install instructions in `README.md`.** Concrete `cp -r` commands for installing every skill into `~/.claude/skills/`, plus a note on when to prefer the folder copy over the `.skill` archives. (Partial ROADMAP 1.5 — canonical path picked, exact commands given.)
- **`.gitignore` guards** for `/.idea/` and `/main.py` to prevent JetBrains/PyCharm noise from re-entering the repo, mirroring the existing root `/.obsidian/` guard.

(ROADMAP 1.6)
